# Project Conventions

## Language

- Use English only in source code, comments, documentation, configuration, commit messages, and user-facing text stored in the repository.
- Do not add Russian or any other non-English natural language text to project files.

## Comments

- Do not leave commented-out code in the repository.
- Remove lines that only disable code instead of deleting or rewriting it properly.
- If an explanation is necessary, prefer clear code over comments.
